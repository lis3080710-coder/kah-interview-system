# 🚀 빠른 시작 가이드

## ⚡ 5분 안에 시작하기

### 1단계: 프로젝트 설치 (1분)

```bash
cd kah-interview-system
npm install
```

### 2단계: Supabase 설정 (2분)

#### A. 데이터베이스 스키마 실행
1. https://supabase.com/dashboard 접속
2. 프로젝트 선택
3. 왼쪽 메뉴에서 **SQL Editor** 클릭
4. `supabase-schema.sql` 파일 열기
5. 전체 내용 복사
6. SQL Editor에 붙여넣고 **Run** 클릭
7. "Success. No rows returned" 메시지 확인

#### B. 환경 변수 확인
`.env.local` 파일이 이미 준비되어 있습니다!
```
NEXT_PUBLIC_SUPABASE_URL=https://kldmiyyjjlsysjevmblz.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
```

### 3단계: 실행 (30초)

```bash
npm run dev
```

브라우저에서 http://localhost:3000 열기

## ✅ 설치 확인

### 1. 홈 화면 확인
- KAH 로고 보임 ✅
- 타이머 있음 ✅
- 사이드바 있음 ✅

### 2. 기능 테스트

#### Test 1: 신규 지원자 추가
1. **"+ 신규 지원자"** 버튼 클릭
2. 이름 입력: "테스트"
3. **"✓ 평가 저장"** 버튼 클릭
4. 오른쪽 상단에 초록색 토스트 메시지 확인
5. 사이드바에 "테스트" 지원자 나타남 ✅

#### Test 2: 평가하기
1. 사이드바에서 "테스트" 클릭
2. 점수 입력:
   - 성실성: 3
   - 협조성: 2
   - 표현력: 3
3. 태그 선택: "논리정연함" 클릭
4. **"✓ 평가 저장"** 클릭
5. 총점 자동 계산됨 ✅

#### Test 3: 평균 점수 확인
1. 새 브라우저 창 열기 (시크릿 모드)
2. http://localhost:3000 접속
3. "테스트" 지원자 클릭
4. 다른 점수 입력
5. 저장
6. 사이드바에서 평균 점수 확인 ✅
   - 예: "평균: 25점 (2명)"

## 🔍 문제 해결

### "평가가 저장되지 않아요"
```bash
# 1. 콘솔 확인 (F12)
# 2. Network 탭에서 에러 확인
# 3. Supabase 연결 확인

# Supabase에서 확인:
# Dashboard → Table Editor → evaluations 테이블 확인
```

### "순위가 안 바뀌어요"
```bash
# 브라우저 새로고침 (F5)
# 다른 지원자 클릭 후 다시 클릭
```

### "데이터가 안 보여요"
```sql
-- Supabase SQL Editor에서 실행:
SELECT * FROM candidates;
SELECT * FROM evaluations;

-- 데이터 없으면 스키마 다시 실행
```

## 📚 자주 묻는 질문

**Q: 면접관 ID는 어떻게 관리되나요?**
A: LocalStorage에 자동 생성됩니다. 브라우저마다 다른 ID를 가집니다.

**Q: 실제 PDF 파싱도 되나요?**
A: 현재는 시뮬레이션입니다. 실제 구현은 추가 개발이 필요합니다.

**Q: 몇 명까지 동시에 평가할 수 있나요?**
A: 제한 없습니다. Supabase 무료 플랜 한도까지 가능합니다.

**Q: 데이터는 어디에 저장되나요?**
A: Supabase (PostgreSQL) 클라우드 데이터베이스에 저장됩니다.
